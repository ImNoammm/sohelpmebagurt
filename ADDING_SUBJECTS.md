# Adding a New Subject to SoHelpMeBagrut

This guide is for the AI assistant that will help add a new subject. Follow every section in order.

---

## 1. Information to gather from the user first

Before writing any code, ask the user for:

1. **Subject name in English** — used for the folder name (e.g. `Math`, `Physics`, `English`)
2. **Subject name in Hebrew** — shown on the site (e.g. `מתמטיקה`, `פיזיקה`, `אנגלית`)
3. **Exam number(s)** — the שאלון codes (e.g. `035001`)
4. **Language variants?** — does this subject have multiple language versions (like CS has Java/C#)? If yes, what are they?
5. **Ministry of Education bagruyot API URL** — ask the user to open this URL in their browser, replace the subject code, and paste the JSON:
   `https://meyda.education.gov.il/bagmgr/Ajax.ashx?search=1&sheelon=&miktzoa=SUBJECT_CODE&safa=1&pagesize=100&page=1&csrt=TOKEN`
   The `miktzoa` value is the subject code (e.g. 899 = CS, 035 = Math). The user can find it by opening the search on meyda.education.gov.il and inspecting the network request.
6. **Icon/color for the site card** — suggest a fitting emoji or color, let user confirm.

---

## 2. Folder structure to create

```
subject/{SubjectName}/
├── shared/
│   └── base.md          # Shared instructions fetched by all skill files
└── {variant}/
    └── skill.md         # One folder per language/variant (or just "default" if no variants)
```

**If no language variants** (e.g. Math), create:
```
subject/Math/
├── shared/base.md
└── default/skill.md
```

**If language variants exist** (like CS), create:
```
subject/Physics/
├── shared/base.md
├── hebrew/skill.md
└── arabic/skill.md
```

The `bagrut/` subfolder is created automatically by the GitHub Action — do NOT create it manually.

---

## 3. Write `shared/base.md`

Model it after `subject/ComputerScience/shared/base.md`. Must include:

- **Greeting section** — same format: say "🎓 **SoHelpMeBagrut loaded!**" on first message
- **Role** — describe the AI's role for this specific subject
- **Behavior rules** — how to present questions, prefer recent years, wait for student to attempt
- **Style** — emojis, diagrams, no walls of plain text
- **Bagruyot Files section** — leave empty initially with the comment:
  `<!-- URL list is auto-generated by scripts/sync_bagrut.py — do not edit manually -->`
  The GitHub Action will populate it after the first sync.
- **הקלות section** — ask the user if there are any topics excluded from the current מיקוד

Base URL for exam files:
`https://imnoammm.github.io/sohelpmebagurt/subject/{SubjectName}/bagrut/{exam_number}/{year}_{moed}_exam.txt`

---

## 4. Write `skill.md` for each variant

Model it after `subject/ComputerScience/csharp/skill.md`. Must include:

```markdown
# {Subject} Bagrut Teacher — Model Instructions

> **Start by fetching the shared base instructions:**
> `WebFetch("https://imnoammm.github.io/sohelpmebagurt/subject/{SubjectName}/shared/base.md")`
> Apply everything there, then apply the subject-specific rules below.

---

## Behavior — {Variant} Specific

(any variant-specific rules, terminology, formatting conventions)

---

## Key Concepts / Formula Sheet

(the equivalent of the API reference in CS — formulas, theorems, vocabulary, etc.)
```

The skill.md URL (what gets copied on the site) will be:
`https://imnoammm.github.io/sohelpmebagurt/subject/{SubjectName}/{variant}/skill.md`

## 5. Update the GitHub Action (`scripts/sync_bagrut.py`)

The sync script is currently hardcoded for CS (`miktzoa=899`). To add a new subject:

1. Open `scripts/sync_bagrut.py`
2. Find the `fetch_all_exams()` function
3. The script currently calls the API with `miktzoa=899` — make it configurable by adding a `SUBJECTS` list at the top of the file:

```python
SUBJECTS = [
    {"miktzoa": "899", "folder": "ComputerScience"},
    {"miktzoa": "035", "folder": "Math"},   # add new subject here
]
```

4. Loop over `SUBJECTS` in `main()` and call `fetch_all_exams()` + `sync_exams()` for each
5. Update `BAGRUT_DIR` and `GITHUB_BASE` to be per-subject inside the loop
6. Also update `regenerate_url_lists()` to handle multiple subjects' `base.md` files

Also update `.github/workflows/sync-bagrut.yml` if any new system dependencies are needed.

## 6. Update `index.html` (the site UI)

Find the `<div class="subjects-grid">` section. Add a new card for the subject.

**Card template for an active subject (with one variant):**
```html
<div class="subject-card active">
  <div class="card-top">
    <div class="card-icon" style="background:#COLOR;">EMOJI</div>
    <span class="badge badge-live">זמין עכשיו</span>
  </div>
  <div class="card-title">{Hebrew Subject Name}</div>
  <div class="card-subtitle">{exam numbers} · {variant}</div>
  <div class="card-url-row">
    <code>imnoammm.github.io/sohelpmebagurt/subject/{SubjectName}/{variant}/skill.md</code>
    <button class="copy-btn" onclick="copyUrl(this, 'https://imnoammm.github.io/sohelpmebagurt/subject/{SubjectName}/{variant}/skill.md')">העתק</button>
  </div>
</div>
```

**Card template for coming soon:**
```html
<div class="subject-card coming-soon">
  <div class="card-top">
    <div class="card-icon" style="background:#COLOR;">EMOJI</div>
    <span class="badge badge-soon">בקרוב</span>
  </div>
  <div class="card-title">{Hebrew Subject Name}</div>
  <div class="card-subtitle">{description}</div>
</div>
```

Also update the "coming soon" card for this subject if it already exists (Math, Physics, English cards are already on the site as coming-soon — just convert the existing card to active).

---

## 7. Commit and push everything

```bash
git add subject/{SubjectName}/ index.html scripts/sync_bagrut.py .github/workflows/
git commit -m "Add {SubjectName} subject"
git push
```

Then trigger the sync workflow to populate the bagrut files:
```bash
curl -s -X POST \
  -H "Authorization: token GITHUB_PAT" \
  -H "Accept: application/vnd.github+json" \
  https://api.github.com/repos/ImNoammm/sohelpmebagurt/actions/workflows/sync-bagrut.yml/dispatches \
  -d '{"ref":"main"}'
```
